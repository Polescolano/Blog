Create a full App development, without knowing how to code, ideal for digitizing your organization!

Introduction

Let's start with a brief introduction. What is Power Apps?

Power Apps is a suite of apps, services, and connectors that lets you create custom business apps without coding (or coding a little; Excel like). It provides a rapid development environment where you can build apps according to your business needs. With Power Apps, you can build professional solutions using AI-powered development, prebuilt templates, and connectors to hundreds of data sources and systems. You can solve countless business problems with Power Apps, whether it's finance, sales and marketing, human resources, operations, or frontline workers.

Power Apps can connect to data you're already using. It can provide you a front-end app to interact with your data on any device, web browsers, mobile devices and tabets.

The best thing about Power Apps is that you don't need coding experience to begin using it. The interface in the Power Platform studio enables you to produce solutions that can transform your manual business processes to digital ones, and start digitizing your organization.

Starting Power Apps - The enviroment

To get started using Power Apps, you need to sign in to your organizational Microsoft 365 Account. Then from your web browser, go to make.powerapps.com. This web address takes you to the Power Apps Maker Portal where you can begin creating your app.

‚ö†Ô∏è Notice a menu down the left-hand side of your screen, and a Copilot entry field in the top-center of the screen. Copilot provides a way to describe what you want your app to do in everyday language. However, I do not recommend using it if you are new here! there's an add-on cost. ‚ö†Ô∏è

To get started with building an app on any data source, you select the Create button from the left-hand menu. Power Apps gives you shortcuts to start your app from some of the most common data sources including SharePoint and Excel, both of which are included in your organizational Microsoft 365 Account.

There are links to instructional videos for the most common ways to start, and if you continue to scroll down the Create page you see templates that are available to use. Templates are a great way to see how an app can behave with data, however, you don't want to use any of these templates with your own data. As you look through the templates, notice that some of them are Model-driven apps. Model-driven apps are form-based apps and aren't covered in this learning module. If you decide to try an app, make sure that it's a Canvas app.

üí° Canvas app allow you to create freely both in front-end design and back-end code. 
Model-driven apps are form-based, nice for creating a quick CRM(Dynamics) add-on for exemple, 
but it's limited.

In the Learn tab, you find links to available Microsoft training and News, Blogs, Discussion forum and a host of how-to articles and instructional videos.

The Apps tab provides a listing of all your apps and buttons to start a new app, like the ones in the Create tab.

Well then, please click on the New app button ‚Üí select Canvas app ‚Üí select name (like Test_firstApp) and Format: tablet ‚Üí Start!üöÄ

Now you're in the main Power Apps create page, üßò relax, isn't a big deal really! üòä

The best way to learn how to use it is by touching it all. But let me drive you through the interface:

At the left vertical bar:
  • Layers icon: will show you the Tree view, all the elements in the app, such as a input text component, image, gallery, screens, etc.
  • Plus icon: will show you the list of elements you can add at the app. Go have a look.
  • Database icon: will show you the data connectors you have and let you add more.
  • Media icon: here you'll upload all the media, to later a add it to a any element.
  • Configuration icon: here you'll manage all the app conf.
At the top bar:
  • top right icons: this area allow you to un-do/re-do, play the app (view how it would look), share the app to cooperate.
  • top left options: here are the tipical tool options, just check it at need.
  • middle horizontal bar: this for adding a new screen(check options) and the text formating options.
  • bottom bar: This is an important one, first; every component at the app, have this attributes(fill, onClick, color, visible, etc.) to edit and here is where the code is put. How? with the following FX text input.
At the right side:
  • Propesties: Here you'll find the main components attributes, with a quick access.
  • Advanced: More components options.
  • Ideas: just play with it, spoiler alert: not commonly used.

Starting Power Apps - Developing

Let's add some data source first. The easiest and most used databases for Power Apps is Sharepoint List because it's very well integrated in the microsoft enviroment, it's relational database, cloud serverless and my favorite: it's free. But if it's your first time in Power Platform, I'll share with you how to upload an Excel to SharePoint List, this Lists are like a database table. Please follow these easy steps: How to create a Sharepoint List from an Excel.

Now that we have that database, let's create our 1st screen with a gallary table:
  1. Go to insert ‚Üí select Gallery. You'll see appeared a Lorem ipsum(sample) gallery.
  2. This gallery will ask you to select the data you want to show, select the Sharepoint list that you've created.
  3. Alright the table now is ugly, worry not:

  4. Now you have the gallary nice created, and you can continue playing with it, for exemple, selecteing the order of the data. So with the gallary selected, go to the top ‚Üí FX input text, you should see you data table name. ‚Üí Edit that text and write this: "Sort( [database_name], [column_name], descending )" and whatch how the gallary change.
